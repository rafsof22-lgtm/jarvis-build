# Universal Model Selector, Parallel Thinking and ChatHub Intake

Load this reference whenever Jarvis exposes a model selector, a multi-model panel, ChatHub source files, response consolidation or a request for zero-loss multi-LLM synthesis.

## Universal selector

Use one shared selector contract across Jarvis Pop, Command Centre, framework/spec/workflow/stack/orchestrator/Skill/agent editors, prompt library, source analyst, evaluation console and deployment wizard.

Enumerate concrete model IDs from each available route's existing default-model environment variable plus `JARVIS_<ROUTE_NAME>_MODELS`. Show every unique configured model. Never invent a model ID. A configured route with no model inventory is a visible blocker.

## Parallel Thinking button

Expose `Parallel Thinking · Up to 8 LLMs` only when at least two concrete models are connected. Permit manual selection of 2–8 models and automatic best-fit selection subject to privacy, capability, cost, context and provider policy.

Assign distinct roles when useful: primary reasoner, source researcher, systems architect, domain specialist, safety critic, cost critic, independent verifier and final synthesiser.

Preserve every raw response separately with model ID, route, role, timestamp, response hash, citations and claim IDs. Consolidation is a separate derived artifact. Use `selector_id + response_sha256` as the proof key so identical answers from different models remain independently accounted.

Require a claim/evidence matrix, agreement register, contradiction register, unique-insight register, omission register and unresolved-conflict section. Majority agreement is not proof. Prefer current authoritative evidence, safety, compliance and explicit uncertainty.

Do not promise an ultimate, perfect, zero-error or universally complete answer. The strongest permitted label is `NO_KNOWN_GAPS_WITHIN_VERIFIED_PANEL_SCOPE` after every selected response and required section is proven included.

## High-risk domains

Health, legal, tax, finance, live trading, personal-data export, credentials, production and destructive actions require specialist policy and qualified human or professional review. Parallel model agreement cannot replace those gates.

## ChatHub source intake

Preserve each raw file by immutable pointer, exact SHA-256, byte/line/word counts and message coordinates. Parse explicit user/system/model boundaries. Treat user messages as requirement candidates. Treat model responses as proposals or source claims, never automatic approvals.

Every parsed message must receive an authority class, exact-duplicate lineage, module routes, risk flags and one disposition: requirement candidate, reference/delta review, quarantine, duplicate-with-lineage or explicit exclusion with reason.

Quarantine guaranteed cures/profits, experimental medical execution, access-control bypass, unsupported platform claims and credential exposure. Preserve them as evidence but do not activate them.

Raw health or third-party data should remain in the authorised source store where possible. Commit derived manifests and hashes rather than duplicating sensitive source text into the repository.
