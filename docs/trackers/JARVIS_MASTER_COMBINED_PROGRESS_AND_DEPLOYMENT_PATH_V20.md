## 1A. Sidebar-chat definition and capture test

For this tracker, **sidebar chats** means conversations visibly listed under the current Jarvis ChatGPT Project in the ChatGPT Project sidebar. The historical account export, semantic Jarvis candidates, named domain denominators and shared links are separate source classes and must not be substituted for the live sidebar denominator.

A sidebar chat is **fully captured** only when all available user, assistant, system and tool messages are preserved with source pointers; attachments are recovered or gap-logged; branches and parent links are reconciled; requirements, decisions, conflicts and open loops are mapped; and the resulting records link to Jarvis modules, artifacts, tests/waivers, evidence, runtime state and rollback. Merely seeing a title, summary, URL or partial context is not full capture.

**Current proven answer:** authoritative sidebar total = `UNKNOWN`; fully captured sidebar chats = `0 PROVEN`. The runtime currently exposes the active conversation and bounded prior project context, but it cannot enumerate the complete Project sidebar or certify end-to-end reconstruction of any sidebar chat.
