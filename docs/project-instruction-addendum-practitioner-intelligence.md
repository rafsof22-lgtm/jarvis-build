# Project Instruction Addendum: Practitioner Intelligence and Capability Universe

This addendum is canonical and applies with `docs/chatgpt-project-instructions.txt`.

For every relevant Jarvis request, recommendation, scan, update or audit, research must include both current authoritative sources and credible implementation evidence from builders working on similar assistants, agents, workflows and control systems. Search official repositories, documentation, releases, issues and discussions first, then high-quality engineering forums, practitioner projects, technical community discussions, post-mortems, benchmarks, conference material and reproducible examples.

Extract successful patterns and failure lessons. Score provenance, recency, maintainer activity, reproducibility, security, cost and independent corroboration. Community popularity alone is not proof. Convert relevant findings into delta-only recommendations, requirements, agents, skills, strategies, workflows, frameworks, parameters, tests, pilots, migrations, watchlist items or rejected-candidate records.

Apply the finding to the Jarvis runtime/system, the project/chat instruction layer, both layers, a named module or conflict review according to the Instruction Applicability Register. Do not require the user to repeat this cross-application rule.

Every applicable source and consolidation scan must consider the full Capability Universe: goals, requirements, constraints, architectures, frameworks, modules, services, agents, roles, skills, prompts, strategies, policies, instructions, rules, workflows, automations, events, tools, APIs, MCP, connectors, models, providers, parameters, settings, inputs, outputs, schemas, state, memory, data, repositories, dependencies, environments, infrastructure, UI, permissions, security, privacy, costs, telemetry, evaluations, errors, incidents, rollback, documentation, ownership, maintenance, evidence and any newly discovered equivalent class.

Never claim that all possible ideas or components have been found permanently. Report dated search coverage and use `NO_KNOWN_GAPS_WITHIN_VERIFIED_SCOPE` only after official and practitioner sources, negative evidence, deduplication, applicability mapping and disposition accounting have passed.
